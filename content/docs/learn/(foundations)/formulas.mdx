If you have never opened a spreadsheet, here is the whole idea in one
sentence: a spreadsheet is a grid of boxes (called **cells**), and each box
holds either a plain value ‚Äî like `42` or `"Paid"` ‚Äî or a **formula** that
computes a value from other values.

A formula always starts with an equals sign:

```
=3+4
```

When a spreadsheet sees that leading `=`, it doesn't store the text `3+4` ‚Äî
it computes the answer, `7`, and shows you that instead. Everything else in
this guide builds on that single trick.

## Evaluating a formula with TrueCalc

TrueCalc is a formula engine without the grid: you hand it a formula, it
hands back the computed value. You always start by creating an **engine**
for an explicit conformance target ‚Äî the spreadsheet product whose behavior
you want, exactly. (Why you must choose is the subject of the
[dialects chapter](/learn/dialects).)

```ts test
import { createEngine } from '@truecalc/core';
import assert from 'node:assert/strict';

// Always name your target explicitly. 'google-sheets' is the
// conformance target verified by the Google Sheets fixtures.
const engine = createEngine('google-sheets');

// + adds two numbers. Sheets exposes the same operation as the ADD
// function ‚Äî fixture: google_sheets/operator.tsv ‚Äî
// "basic addition of two integers" (=ADD(3,4) ‚Üí 7) and
// "GS-only: no equivalent in Excel (use + operator)".
const result = engine.evaluate('=3+4', null);
assert.deepEqual(result, { type: 'number', value: 7 });
```

Every result has a `type` and (usually) a `value` ‚Äî the
[values and types chapter](/learn/values-and-types) tours all of them.

## The operators

Formulas combine values with **operators**. Google Sheets defines each
operator's behavior; it also exposes each one as an equivalent function,
which is how the conformance fixtures pin the behavior down:

| Operator | Meaning            | Sheets function form | Fixture (google_sheets/‚Ä¶)                                       |
| -------- | ------------------ | -------------------- | --------------------------------------------------------------- |
| `+`      | addition           | `ADD`                | operator.tsv ‚Äî "basic addition of two integers"                 |
| `-`      | subtraction        | `MINUS`              | operator.tsv ‚Äî `=MINUS(‚Ä¶)` rows                                 |
| `*`      | multiplication     | `MULTIPLY`           | operator.tsv ‚Äî "basic multiplication"                           |
| `/`      | division           | `DIVIDE`             | operator.tsv ‚Äî `=DIVIDE(‚Ä¶)` rows                                |
| `^`      | exponentiation     | `POW`                | operator.tsv ‚Äî "2 to the power of 3 is 8"                       |
| `%`      | percent (postfix)  | `UNARY_PERCENT`      | operator.tsv ‚Äî "50 as percent equals 0.5"                       |
| `&`      | text concatenation | `CONCAT`             | text.tsv ‚Äî "CHAR concatenated with string" (`=CHAR(72)&"ello"`) |
| `=` `<>` `>` `>=` `<` `<=` | comparisons | `EQ`, `NE`, `GT`, `GTE`, `LT`, `LTE` | operator.tsv ‚Äî "equal numbers returns TRUE" and friends |

Each line below is executed against the pinned `@truecalc/core` on every CI
run ‚Äî the text after `// =>` is the asserted result:

```formula test
# + and - ‚Äî fixtures: operator.tsv "basic addition of two integers", "=MINUS" rows
=3+4 // => 7
=10-3 // => 7
# ^ ‚Äî fixture: operator.tsv "2 to the power of 3 is 8" (=POW(2,3))
=2^3 // => 8
# % ‚Äî fixture: operator.tsv "50 as percent equals 0.5" (=UNARY_PERCENT(50))
=50% // => 0.5
# & joins text ‚Äî fixture: text.tsv "CHAR concatenated with string"
=CHAR(72)&"ello" // => Hello
# comparisons return TRUE/FALSE ‚Äî fixtures: operator.tsv "equal numbers returns TRUE",
# logical.tsv "comparison as condition: 5>3 is true"
=5=6 // => FALSE
=5>3 // => TRUE
```

## Parentheses group work

Just like in school arithmetic, parentheses decide what happens first:

```formula test
# Fixture: operator.tsv ‚Äî "nested ADD inside MULTIPLY"
# (=MULTIPLY(ADD(2,3),ADD(1,4)) ‚Üí 25)
=(2+3)*(1+4) // => 25
```

## Formulas can decide things

Operators feed naturally into **functions** ‚Äî named operations like `IF`,
which picks one of two values based on a condition:

```formula test
# Fixture: logical.tsv ‚Äî "comparison as condition: 5>3 is true"
=IF(5>3,"greater","not") // => greater
```

Functions are the next chapter's subject ‚Äî TrueCalc implements 484 of them.

## A note on the leading =

Inside a spreadsheet cell, the `=` is what distinguishes a formula from
plain text. TrueCalc's API accepts formulas with or without it; this guide
always writes the `=` so examples can be pasted straight into a real
spreadsheet.

## What you learned

- A formula starts with `=` and computes a value.
- You always evaluate against an explicit engine: `createEngine('google-sheets')`.
- Operators: `+ - * / ^ %` for numbers, `&` for text, `= <> > >= < <=` for comparisons.
- Parentheses group sub-expressions.
